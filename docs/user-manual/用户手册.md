## 1. 系统简介

### 1.1 系统概述

BankShield是一个专业的银行数据安全管理平台，提供数据加密、访问控制、审计追踪、敏感数据识别与脱敏等核心功能，确保银行数据全生命周期的安全性。

### 1.2 核心功能

- **数据加密管理**: 支持国密SM2/SM3/SM4算法和国际标准加密算法
- **细粒度访问控制**: 基于RBAC的权限管理体系
- **实时审计追踪**: 全链路操作日志记录与分析
- **敏感数据识别**: 自动发现和分类敏感数据
- **智能数据脱敏**: 动态和静态数据脱敏
- **安全态势可视化**: 实时安全监控与告警
- **合规性检查**: 符合金融行业标准规范

### 1.3 系统访问

**访问地址**: https://bankshield.com

**默认账号**:
- 超级管理员: admin / Admin@123
- 安全管理员: security / Security@123
- 普通用户: user / User@123

**浏览器要求**:

- Firefox 88+
- Edge 90+
- Safari 14+

## 2. 快速入门

### 2.1 登录系统

1. 打开浏览器，访问系统地址
2. 输入用户名和密码
3. 点击"登录"按钮
4. 首次登录需要修改密码

![登录页面](../images/login.png)

### 2.2 系统首页

登录成功后进入系统首页，可以看到：

- **顶部导航栏**: 系统菜单、用户信息、消息通知

## 3. 用户管理

### 3.1 用户列表

**路径**: 系统管理 > 用户管理

**功能说明**:
- 查看所有用户信息
- 搜索和筛选用户
- 新增、编辑、删除用户
- 重置用户密码
- 启用/禁用用户

**操作步骤**:

1. 进入用户管理页面
2. 可以通过用户名、邮箱等条件搜索
3. 点击"新增"按钮创建新用户
4. 点击"编辑"按钮修改用户信息
5. 点击"删除"按钮删除用户（需确认）

### 3.2 新增用户

1. 点击"新增用户"按钮
2. 填写用户信息：
   - 用户名（必填，唯一）
   - 真实姓名（必填）
   - 邮箱（必填，唯一）
   - 手机号
   - 所属部门
   - 用户角色
3. 点击"确定"保存

**注意事项**:
- 用户名只能包含字母、数字、下划线
- 邮箱格式必须正确
- 初始密码为：User@123

### 3.3 编辑用户

1. 在用户列表中找到目标用户
2. 点击"编辑"按钮
3. 修改用户信息
4. 点击"确定"保存

### 3.4 重置密码

1. 在用户列表中找到目标用户
2. 点击"重置密码"按钮
3. 确认操作
4. 密码将重置为：User@123

## 4. 角色管理

### 4.1 角色列表

**路径**: 系统管理 > 角色管理

**功能说明**:
- 查看所有角色
- 新增、编辑、删除角色
- 配置角色权限
- 查看角色关联的用户

### 4.2 新增角色

1. 点击"新增角色"按钮
2. 填写角色信息：
   - 角色名称（必填）
   - 角色编码（必填，唯一）
   - 角色描述
   - 数据权限范围
3. 选择菜单权限
4. 点击"确定"保存

**数据权限范围**:
- 全部数据：可以查看所有数据
- 本部门数据：只能查看本部门数据
- 本人数据：只能查看自己的数据

### 4.3 配置权限

1. 在角色列表中找到目标角色
2. 点击"权限配置"按钮
3. 勾选需要授予的菜单权限
4. 点击"确定"保存

## 5. 数据加密

### 5.1 密钥管理

**路径**: 安全管理 > 密钥管理

**功能说明**:
- 查看密钥列表
- 生成新密钥
- 密钥轮换
- 密钥启用/禁用

### 5.2 生成密钥

1. 点击"生成密钥"按钮
2. 填写密钥信息：
   - 密钥名称（必填）
   - 密钥类型（SM2/SM4/AES/RSA）
   - 密钥用途
   - 轮换周期（天）
3. 点击"确定"生成

**密钥类型说明**:
- **SM2**: 国密非对称加密算法
- **SM4**: 国密对称加密算法
- **AES**: 国际标准对称加密
- **RSA**: 国际标准非对称加密

### 5.3 密钥轮换

1. 在密钥列表中找到目标密钥
2. 点击"轮换"按钮
3. 确认操作
4. 系统自动生成新密钥并替换旧密钥

**注意**: 密钥轮换会影响使用该密钥加密的数据，请谨慎操作。

### 5.4 数据加密

1. 进入需要加密的数据页面
2. 选择要加密的数据
3. 点击"加密"按钮
4. 选择加密密钥
5. 确认加密

## 7. 数据脱敏

### 7.1 脱敏规则

**路径**: 安全管理 > 数据脱敏 > 脱敏规则

**功能说明**:
- 查看脱敏规则列表
- 新增、编辑、删除规则
- 启用/禁用规则
- 测试规则效果

### 7.2 新增规则

1. 点击"新增规则"按钮
2. 填写规则信息：
   - 规则名称（必填）
   - 规则类型（掩码/替换/加密/哈希）
   - 数据类型（手机号/身份证/银行卡等）
   - 脱敏字符（默认*）
   - 保留前缀长度
   - 保留后缀长度
   - 优先级
3. 点击"确定"保存

**脱敏类型说明**:
- **掩码**: 用*号替换部分字符
- **替换**: 用指定字符替换
- **加密**: 使用加密算法加密
- **哈希**: 使用哈希算法处理

### 7.3 测试规则

1. 在规则列表中找到目标规则
2. 点击"测试"按钮
3. 输入测试数据
4. 查看脱敏效果
5. 确认无误后启用规则

## 8. 安全扫描

### 8.1 创建扫描任务

**路径**: 安全管理 > 安全扫描 > 扫描任务

**操作步骤**:

1. 点击"新建任务"按钮
2. 填写任务信息：
   - 任务名称（必填）
   - 扫描类型（代码扫描/依赖扫描/配置扫描）
   - 目标路径
   - 扫描规则
3. 点击"确定"创建
4. 点击"执行"按钮开始扫描

### 8.2 查看扫描结果

1. 等待扫描完成
2. 点击"查看结果"按钮
3. 查看发现的漏洞：
   - 漏洞名称
   - 严重程度
   - 漏洞位置
   - 修复建议
4. 可以导出扫描报告

### 8.3 漏洞管理

**路径**: 安全管理 > 安全扫描 > 漏洞管理

**功能说明**:
- 查看所有漏洞
- 分配漏洞处理人
- 标记漏洞状态
- 查看漏洞详情

**漏洞状态**:
- **未解决**: 新发现的漏洞
- **处理中**: 正在修复
- **已解决**: 已修复完成
- **误报**: 确认为误报

## 11. 常见问题

### 11.1 登录相关

**Q: 忘记密码怎么办？**

A: 请联系系统管理员重置密码，或使用"忘记密码"功能通过邮箱重置。

**Q: 登录提示账号被锁定？**

A: 连续输错密码5次会被锁定30分钟，请等待解锁或联系管理员。

### 11.2 权限相关

**Q: 为什么看不到某些菜单？**

A: 可能是权限不足，请联系管理员分配相应权限。

**Q: 如何申请新的权限？**

A: 联系部门负责人或系统管理员申请。

### 11.3 操作相关

**Q: 如何导出数据？**

A: 在列表页面点击"导出"按钮，选择导出格式即可。

**Q: 上传文件有大小限制吗？**

A: 默认限制为10MB，如需上传更大文件请联系管理员。

### 11.4 性能相关

**Q: 页面加载很慢怎么办？**

A: 
1. 检查网络连接
2. 清除浏览器缓存
3. 尝试刷新页面
4. 如问题持续，联系技术支持

## 12. 最佳实践

### 12.1 密码安全

- 定期修改密码（建议每3个月）
- 使用强密码（包含大小写字母、数字、特殊字符）
- 不要与他人共享账号
- 不要在公共场所登录系统

### 12.2 数据安全

- 敏感数据必须加密存储
- 定期检查数据访问日志
- 及时处理安全告警
- 遵守数据分类分级规范

### 12.3 操作规范

- 重要操作前先备份数据
- 批量操作前先测试
- 定期查看审计日志
- 及时更新系统版本
